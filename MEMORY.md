# Genesis AI Memory

## Назначение

Этот документ хранит долгосрочную память Genesis AI: подтверждённые сведения, важные решения и знания, которые должны сохраняться между сессиями.

---

# Основные сведения

Название: Genesis AI

Текущая фаза: Фаза 2 — Архитектура системы.

Главная цель: Создать управляемую ИИ-команду, способную проектировать, разрабатывать и сопровождать программные продукты.

# Участники

## CEO

Человек — владелец проекта. Принимает окончательные решения.

## COO

ChatGPT. Управляет процессами и координирует работу.

## CTO

ChatGPT — временно. Отвечает за архитектуру и анализ рисков.

## GitHub Engineer

GitHub Copilot. Выполняет текущую работу с кодом и файлами (решение DR-0002).

## Developer

Cursor. Выполняет сложные архитектурные работы.

## Repository

GitHub — единый источник истины.

# Завершённые циклы

## Цикл 1: Основание

- фундамент проекта создан;
- Git и GitHub настроены;
- Конституция, роли, принципы, стандарты определены;
- видение и Roadmap созданы.

## Цикл 2: Структура

- репозиторий разделён на смысловые области;
- управляющие и стратегические документы организованы.

## Цикл 3: Decision Records

- система Decision Records создана;
- DR-0001 (архитектура репозитория) принято;
- DR-0002 (назначение GitHub Engineer) принято.

# Активные задачи

- создать критерии оценки кандидатов на постоянного CTO;
- подготовить тестовое задание для испытания;
- провести испытание моделей (ChatGPT, Claude и др.);
- оформить DR-0003 по результатам испытания.

---

# Принципы

- каждая завершённая задача оставляет проверяемый результат и знание;
- если изменение не в GitHub, оно не часть Genesis AI;
- агент выполняет только поставленную задачу, предлагая дополнительную работу как отдельную задачу.
