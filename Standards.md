# Genesis AI Standards

## Назначение

Этот документ определяет единые стандарты работы проекта.

Любой участник команды должен соблюдать эти правила независимо от своей роли.

---

# Основной принцип

Простые решения предпочтительнее сложных.

Сложность должна появляться только тогда, когда она действительно необходима.

---

# Работа с изменениями

Перед изменением необходимо:

- понять существующее решение;
- определить цель изменения;
- оценить возможные последствия.

После изменения необходимо:

- проверить результат;
- обновить документацию при необходимости;
- сохранить изменения в Git.

---

# Работа с кодом

Код должен быть:

- читаемым;
- предсказуемым;
- минимально сложным;
- легко расширяемым.

Не допускается:

- дублирование логики;
- скрытые зависимости;
- необъяснимые решения;
- временные исправления без пометки.

---

# Работа с документами

Каждый документ должен иметь:

- понятное назначение;
- логичную структуру;
- актуальное содержание.

Документы должны отражать реальное состояние проекта.

---

# Работа с ИИ

ИИ обязан:

- отделять факты от предположений;
- явно сообщать о неопределённости;
- не придумывать отсутствующую информацию;
- объяснять важные решения.

---

# Работа с Git

Каждое значимое изменение должно:

- иметь понятный commit;
- быть сохранено в репозитории;
- иметь возможность восстановления.

История изменений считается частью проекта.

---

# Проверка качества

Перед завершением задачи необходимо убедиться, что:

- задача действительно выполнена;
- результат соответствует цели;
- отсутствуют очевидные ошибки;
- документация обновлена при необходимости.

---

# Развитие проекта

Каждое изменение должно отвечать хотя бы одному из вопросов:

- делает ли систему проще?
- делает ли систему понятнее?
- делает ли систему надёжнее?
- делает ли систему быстрее?
- делает ли систему умнее?

Если ответ отрицательный на все вопросы — изменение требует повторной оценки.
